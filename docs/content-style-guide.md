# Guia de texto do produto

## Objetivo

Padronizar todos os textos visíveis do sistema em português do Brasil formal, claro e consistente.

## Princípios

- Escrever para pessoas não técnicas.
- Preferir frases curtas e diretas.
- Usar linguagem profissional, educada e objetiva.
- Encerrar mensagens completas com ponto final.
- Evitar inglês desnecessário na interface.
- Evitar caixa alta fora de siglas e códigos.

## Capitalização

- Títulos e menus: capitalização natural em português.
- Botões: verbo no infinitivo curto ou ação direta.
- Labels de formulário: substantivo claro e objetivo.
- Status: primeira letra maiúscula, sem caixa alta total.

## Padrões de mensagem

- Erro: “Não foi possível carregar os dados.”
- Sucesso: “As alterações foram salvas com sucesso.”
- Confirmação: “Tem certeza de que deseja excluir este registro?”
- Sessão: “Sua sessão expirou. Faça login novamente.”
- Vazio: “Nenhum item foi encontrado.”
- Validação: “Preencha os campos obrigatórios antes de continuar.”

## Regras de revisão

- Não expor detalhes técnicos ao usuário final.
- Não misturar variações semânticas para o mesmo conceito.
- Revisar sempre acentuação, pontuação e concordância.
- Manter a mesma mensagem equivalente em backend, web e mobile.
